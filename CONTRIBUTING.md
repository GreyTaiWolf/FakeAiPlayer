# 贡献与提交规范

## 中文更新说明

从 2026-07-23 起，本仓库的新提交标题、PR 标题、PR 更新摘要、合并说明、发布说明和 `CHANGELOG.md` 条目使用简体中文。

类名、方法名、协议名、命令、配置键、第三方项目名和错误原文等技术标识可以保留英文。旧提交历史不为满足本规范而改写。

推荐提交标题：

```text
新增：完成 Mission 优先级仲裁
修复：避免重启后恢复预算清零
测试：补充双加载器铁锭黄金链
文档：更新 P3 状态与验证证据
重构：收敛 GoalExecutor 恢复逻辑
```

## 主分支策略

低风险、小范围更新在相关测试通过后可以合入 `main`。以下高风险更新应保留独立分支，通过 PR、Java 21 CI 和双加载器验证后再合并：

- 存档格式、数据迁移或恢复逻辑；
- 网络协议、权限、安全边界或 API Key 处理；
- 导航、Mission/Skill、Task 仲裁和大规模状态机重构；
- 方块破坏、放置、建筑施工等世界修改规则；
- Fabric 与 NeoForge 的加载器适配或发布构建；
- 难以安全回滚的大规模跨模块修改。

## 更新日志

对玩家行为、运行时语义、配置、存档、兼容性或安全有影响的更新必须同步修改 [`CHANGELOG.md`](CHANGELOG.md)。若 PR 不需要更新日志，应在 PR 中写明原因。

更新日志中的测试结论必须绑定具体提交和 CI。CI 尚未结束时只能写“待验证”，不能提前写成“全部通过”。

## 验证

根据改动范围执行 [`README.md`](README.md#构建开发与测试) 和 [`docs/TESTING_AND_EVIDENCE.md`](docs/TESTING_AND_EVIDENCE.md) 中对应的检查。涉及共享业务逻辑时，Fabric 与 NeoForge 必须验证同一语义，不能只用一个加载器的结果代替。

## 安全

不得提交 API Key、访问令牌、个人通信、未脱敏日志、真实服务器地址或不必要的玩家/世界隐私数据。测试夹具与证据包只保留复现问题所需的最小信息。
